# Настройка системы мониторинга ошибок

## Описание

Система мониторинга ошибок автоматически отправляет уведомления об ошибках в Telegram канал для быстрого реагирования на проблемы.

## Настройка

### 1. Создание Telegram канала

1. Создайте новый канал в Telegram
2. Добавьте бота в канал как администратора
3. Получите ID канала:
   ```bash
   curl "https://api.telegram.org/bot<YOUR_BOT_TOKEN>/getUpdates"
   ```
4. Найдите `chat.id` в ответе (для каналов ID начинается с -100)

### 2. Настройка переменных окружения

Добавьте в ваш `.env` файл:

```env
# Error Monitoring Configuration
TELEGRAM_ERROR_BOT_TOKEN=1234567890:ABCdefGHIjklMNOpqrsTUVwxyz
TELEGRAM_ERROR_CHANNEL_ID=-1001234567890
TELEGRAM_ERROR_CHANNEL_NAME=@error_monitoring
TELEGRAM_ERROR_NOTIFICATIONS_ENABLED=true

# Название продакшена (опционально, если не указано - генерируется автоматически)
TELEGRAM_ERROR_ENVIRONMENT_NAME=production-server-1
```

### 3. Проверка системы

```bash
# Проверить статус системы
php artisan error-monitoring:status

# Настроить систему (если нужно)
php artisan error-monitoring:setup
```

## Типы уведомлений

### 🚨 Обычные ошибки
- Ошибки приложения
- Исключения в коде
- Ошибки валидации (в некоторых случаях)

### 🔥 Критические ошибки
- Ошибки базы данных
- Ошибки Redis
- Ошибки памяти
- Fatal errors
- HTTP ошибки 5xx

### ⚠️ Предупреждения
- Потенциальные проблемы
- Нестандартные ситуации
- Системные предупреждения

## Безопасность

Система автоматически очищает чувствительные данные:
- Пароли и токены
- API ключи
- Данные авторизации
- Cookie

## Мониторинг

Все уведомления содержат:
- Время возникновения
- Окружение (production/staging/local)
- Информацию о сервере
- Контекст ошибки
- Трейс выполнения
- Информацию о пользователе (если доступна)

## Отключение

Для отключения системы мониторинга установите:
```env
TELEGRAM_ERROR_NOTIFICATIONS_ENABLED=false
```
